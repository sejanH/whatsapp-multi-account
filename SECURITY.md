## Reporting a Vulnerability

**Important:** Please do not open public GitHub issues for security-related bugs.

If you discover a security vulnerability (e.g., credential leakage, session hijacking, or RCE), please report it privately. You can report vulnerabilities via one of the following:

1. **Email:** Send a detailed report to `sejan@sejan.xyz` .
2. **GitHub Security Advisory:** If enabled, please use the [Draft a new Security Advisory](https://github.com/sejanH/whatsapp-multi-account/security/advisories/new) feature on GitHub.

### What to include in your report:
* A description of the vulnerability.
* Steps to reproduce the issue (Proof of Concept).
* Potential impact if exploited.
* Any suggested fixes (optional).

## Our Commitment

Once a report is received:
1. We will acknowledge receipt of your report within **48 hours**.
2. We will provide a timeline for the fix and keep you updated on progress.
3. We will credit you in our security advisories (if you wish) once the patch is public.

## Responsible Disclosure

We ask that you follow "Responsible Disclosure" principles:
* Give us a reasonable amount of time to fix the issue before sharing it publicly.
* Do not exploit the vulnerability further than necessary to prove its existence.
* Do not access or modify data belonging to other users.
